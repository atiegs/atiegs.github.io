Chapters 9-10

Chapter 9

  1. Structured programming evolved to address what programming practice?
    2. Structured programming addressed one-way code transfer such as GOTO statements. These would often get mixed up creating what was known as “spaghetti code.”

  2. Was structured programming a solution to the problem of software development?
    3. It benefited in a very minimal way, the problem of software development is still a difficult one.

  3. Have any techniques shown to improve the software development process?
    4. Watts Humphrey, the “ultimate” project management disciplinarian, used his “Must. Plan. More!” technique to create successful schedule discipline.

  4. The “waterfall model” of programming was/is popular. What were some problems with this model?
    5. The waterfall model was a programming method in which various tasks would be completed in a cascade. This led to several issues including delays and broken programs.

  5. What are the four tenets of Agile Software Development?
    6. i. Individuals and interactions over processes and tools.
    ii. Working software over comprehensive documentation.
    iii. Customer collaboration over contract negotiation.
    iv. Responding to change over following a plan.

  6. What did a 2004 study find about the development practices of some two hundred software team leaders?
    7. They discovered, much to their dismay, that there was no dominant practice whatsoever.

  7. What is the “Joel Test” and what did he say about Microsoft and the Joel Test.
    8. The Joel test asks 12 questions regarding developmental organization. Any score below 10 is evidence of “serious problems.”

  8. What is Rosenberg’s Law?
    9. Making software is easy until one is required to make it do something new. However, the only software worth making is one that does something new.
