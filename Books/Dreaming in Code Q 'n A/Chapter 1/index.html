Chapter 1

  1. What roles in the Chandler project did Michael Toy, John Anderson, Ted Burgess, Mitchell Kapor, and Lou Montulli hold?
    2. • Michael Toy is the project manager
    • John Anderson is the systems architect and lead coder
    • Ted Burgess is a younger programmer
    • Mitchel Kapor is the founder of OSAF
    • Lou Montulli is a programmer

  2. What is “Bugzilla?”
    3. Bugzilla is a bug tracking system used to identify defects during programming.

  3. What is OSAF?
    4. The Open Source Applications Foundation.

  4. What is the project's name?
    5. The Chandler Project.

  5. What will the software do?
    6. It is a personal information managing software.

  6. What is Toy’s keyword for “black hole” bugs?
    7. “A developer’s nightmare.”

  7. What scared Toy so much about Bug 44?
    8. Bug 44 was contained in wxWidgets, a software the team was using to develop Chandler. In order to fix the issue, they had to either wait for a fix by the coders of wxWidgets or find a workaround themselves.

  8. What did Toy refer to as a “snake?”
    9. “Snakes” are “important problems that we don’t have consensus on how to attack.”

  9. In the software world, what does “slippage” mean?
    10. It is another word for "lateness."

  10. Fredrick Brooks was a programming manager for what software project?
    11. International Business Machines (IBM)

  11. What is Brook's Law?
    12. “Adding manpower to a late software project makes it later.”

  12. Brooks found what % of project time was spent writing code?
    13. He found that 16.67% (or 1/6) of time should be used writing code.

  13. Brooks found what % of project time was for testing and fixing bugs?
    14. 50% of time should be spent testing the program and fixing bugs.

  14. Brooks observed that the unit of effort named “man-month” only applied under what conditions?
    15. Only when the required task could be “partitioned among many workers with no communication among them.”

  15. What is the difference between source code and the program you install (.exe) on your computer?
    16. Source code is the original code written by the programmer(s), it is readable and understandable. An .exe file has been converted into binary and therefore unreadable to humans, only machines.

  16. What is the one “article of faith” that all “open source” or “free” software advocates share?
    17. "Software anyone can tinker with is bound to improve over time in ways that ‘closed’ software can’t match.”

  17. What is the difference between a “good” programmer and a “great” programmer?
    18. Good programmers can write code and great ones can rewrite and reuse it.

  18. Eric Raymond’s book “The Cathedral and the Bazaar” made a distinction between two important project development ideas, briefly contrast them.
    19. • The first method is to build software carefully and quietly until it is time to be released (i.e. a video game designed for a gaming system like the Xbox of PlayStation).
    • The second method is to release early and provide updates as you go (i.e. a smartphone app or a PC video game).

  19. Has “open source” software project development refuted Brooks’ “Mythical Man-Month” concerns?
    20. Open source software revealed a substitute to Brooks’ Law. Open source makes time less important because the work is done by volunteers and therefore more fun and ego driven as opposed to a more corporate approach.

  20. What was Andy Hertzfeld’s input when the Chandler project appeared to have stalled?
    21. His input was to stop designing and begin coding. The sooner they could get exciting work going, the better.
